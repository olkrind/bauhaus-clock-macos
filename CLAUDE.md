# Bauhaus Clock

## What This Is

A macOS screensaver recreation of [bauhausclock.com](https://bauhausclock.com/). Built as a native Swift `.saver` bundle using Core Graphics for all rendering.

## Architecture

```
ClockScreenSaverView.swift     # ScreenSaver.framework entry point, animation loop
  └─ ClockRenderer.swift       # Pure CGContext rendering engine (stateless, hostable in any NSView)
  └─ Palettes.swift            # NSColor palette definitions
  └─ ConfigureSheetController.swift  # Native "Options…" settings sheet (AppKit NSWindow)

preview.swift                  # Standalone NSWindow preview app (reuses ClockRenderer)
  └─ ClockView                 # Thin NSView wrapper calling ClockRenderer.draw()
  └─ SettingsController        # Floating HUD panel (NSPanel) for live dial/lume/size/movement switching
  └─ ToolbarDelegate           # Gear button in window toolbar

CompanionApp/                  # Optional standalone SwiftUI settings app (BauhausClock.app)
ScreenSaverDefaults            # Persistent user preferences
```

Settings are stored via `ScreenSaverDefaults(forModuleWithName:)`:

```swift
"appearance": "night"        // night | day | system (auto via NSAppearance)
"dial":       "Noir"         // active palette (day mode)
"movement":   "Mechanical"   // Quartz | Mechanical | Digital
"size":       "Classic"      // Classic (480px) | Compact (360px)
"lume":       "Tritium Green" // night-mode lume color
"seconds":    true
```

### ⚠️ Settings storage on macOS Sonoma/Sequoia (sandbox)

A legacy `.saver` runs sandboxed inside `legacyScreenSaver.appex`. Its
`ScreenSaverDefaults(forModuleWithName: "com.bauhausclk.BauhausClock")` reads/writes
the **appex container** store, *not* the user's `~/Library/Preferences/ByHost/`:

```
~/Library/Containers/com.apple.ScreenSaver.Engine.legacyScreenSaver/Data/Library/Preferences/ByHost/com.bauhausclk.BauhausClock.<hostUUID>.plist
```

Consequence: an external app (the SwiftUI companion) writing to the user domain is
**invisible** to the saver — it falls back to its registered defaults. The only way OK
actually sticks is to write from *inside* the sandbox → that's why settings live in the
saver's own `configureSheet` (the "Options…" button in System Settings), not (only) in
the companion app.

**Live refresh:** the Options sheet and the preview render run in *separate*
`legacyScreenSaver` processes, so OK can't poke the view directly. On commit the sheet
posts a `DistributedNotificationCenter` notification
(`com.bauhausclk.BauhausClock.configChanged`); `ClockScreenSaverView` observes it and
does `defaults.synchronize()` + `setNeedsDisplay`.

## Build Commands

### Build .saver (no Xcode required)
```bash
./build.sh
# → build/BauhausClock.saver
```

### Companion settings app (optional)
```bash
./build-companion.sh && open build/BauhausClock.app
```
Standalone SwiftUI settings app. NOTE: on Sonoma/Sequoia its writes do **not** reach
the sandboxed saver (see sandbox section above) — prefer the in-saver **Options…** sheet.

### Preview app (fast iteration — no screensaver install needed)
```bash
./build-preview.sh && ./build/BauhausPreview
```

Opens a native NSWindow hosting ClockRenderer directly. Toolbar gear button opens a
floating HUD panel to live-switch dial, size, night mode, lume, and seconds. Also
accepts CLI flags:

```bash
./build/BauhausPreview --dial Turquoise
./build/BauhausPreview --night --lume Amber
./build/BauhausPreview --compact --no-seconds
```

### Install screensaver
```bash
open build/BauhausClock.saver
# macOS prompts to install → "Install for this user only"
# Then: System Settings → Screen Saver → Bauhaus Clock
```

Or copy manually:
```bash
cp -R build/BauhausClock.saver ~/Library/Screen\ Savers/
```

### Xcode project (optional)
```bash
xcodegen generate    # requires: brew install xcodegen
open BauhausClock.xcodeproj
```

## Features

- 16 dial color palettes (White, Turquoise, Glacier, Ocean, Tennis, Signal Blue, Salmon, Yellow, Beige, Pistachio, Lavender, Rose, Sky Blue, Cream, Slate, Noir)
- 3 movement types: Quartz (1Hz tick), Mechanical (4Hz/28,800 bph), Digital (smooth continuous)
- Night mode with 9 lume colors (Tritium Green, Amber, Swiss BGW9, Ice Blue, Red, Lavender, Rose, Seafoam, White)
- Appearance: night / day / system (auto-follows `NSAppearance`)
- Outer minute-numeral ring (60, 05 … 55), tangential with upright bottom half
- Burn-in protection: random ±4px position drift every 45s via `Timer.scheduledTimer` + CGContext translate
- Native **Options…** settings sheet (`configureSheet`) in System Settings — the canonical, sandbox-safe way to change settings
- Classic (480px) / Compact (360px) size toggle
- Second hand toggle

## Design Decisions

### Hands
- **Shape**: Leaf/lozenge bezier path — pointed tip, widest at 38% down, tapered base
- **Rendering**: Outer border layer (scaled 1.12x) + inner metallic fill using 6-stop horizontal linear gradient (`metalEdge → metalMid → metalHi → metalHi → metalMid → metalEdge`)
- **Center highlight**: Faint line for chrome reflection
- **Shadows**: CGContext shadow, dx:1.5 dy:3 blur:3 opacity:0.35

### Hour Indices
- **Shape**: Capsule/stadium rounded rects at all 12 positions (including 12 and 6 o'clock)
- **Rendering**: Same 2-layer border + metallic gradient as hands
- **Shadows** (day): dx:0.5 dy:1.0 blur:1.6 opacity:0.22 (kept subtle so capsules read flat, like the original)
- **Position**: `fR * 0.80` from center, half-length `fR * 0.085`, width 4.5px
  (shortened from 0.11 so the inner tip clears the hour numerals — no overlap)

### Minute Ticks
- 48 lines (skip positions 0, 5, 10, …55 — those are hour indices)
- Gray, 1.3px wide, 50% opacity
- Between `fR * 0.84` and `fR * 0.89`

### Minute Numerals (outer ring)
- "60, 05, 10 … 55" at the 5-minute positions, radius `fR * 0.99`, font `sz * 0.034`
- **Rotated tangentially** around the rim (matches the original) — but the bottom half
  (minutes 20–40, i.e. `90° < angle < 270°`) is flipped 180° so it stays upright,
  not upside down. `drawMinuteNumerals` in `ClockRenderer.swift`.

### Face / Background
- Face color = background color — no visible boundary, no bezel ring, no outer border
- Grain texture: deterministic LCG noise (`x * 1103515245 + 12345`), 3px steps, soft-light blend at 8% opacity. **Cached as a `CGImage` per size** (generated once, not re-filled per frame) — see `grainCache` in `ClockRenderer.swift`.

### Center Hub
- 5-layer concentric rings: outer (8px, `capOuter`), mid (6px, `capMid`), groove (4.5px, `capInner`), bright dot (2.5px, `metalHi`), tiny dark center (1px, `capOuter`)

### Typography
- **Font**: Jost Regular via CoreText — geometric Bauhaus-era typeface. Fallback: Helvetica Neue. (Hour numerals lightened from Medium → Regular to match the original's airier look.)
- **Size**: `sz * 0.046` (hour numerals), `sz * 0.034` (minute numerals)
- **Position**: hour numerals `fR * 0.65` from center (pulled in to clear the shortened capsules)

### Time Calculation (all in degrees)
- Hour hand: `h * 30 + m * 0.5 + s * (0.5/60)`
- Minute hand: `m * 6 + s * 0.1`
- Second hand:
  - Quartz: `s * 6` (discrete 1Hz ticks)
  - Mechanical: `floor((s*1000+ms)/125) * 0.75` (4Hz quantization)
  - Digital: `s * 6 + (ms/1000) * 6` (fully smooth)

### Night Mode
- All black background (`#080808`)
- Hands/indices rendered in lume color with gaussian blur glow (no drop shadows)
- Night palette computed from lume color base

## Palette Structure

Each of the 16 dials defines these color keys in `Palettes.swift`:

```swift
bg          // background + face (same color — no bezel)
idx         // minute tick color
num         // numeral color
sec         // second hand color
metalEdge   // dark edge of metallic gradient
metalMid    // mid tone of metallic gradient
metalHi     // highlight center of metallic gradient
border      // hand/index outer stroke
capOuter    // center hub outer ring
capMid      // center hub mid ring
capInner    // center hub groove ring
```

## Rendering Pipeline (ClockRenderer.swift)

Per frame via `draw(_ rect: NSRect)`:
1. Fill background with palette `bg`
2. Draw grain texture (cached `CGImage`)
3. Calculate hand angles from current time
4. Draw 48 minute ticks (`fR * 0.84–0.89`)
5. Draw 12 hour indices (2-layer capsule, `fR * 0.80`)
6. Draw minute numerals (outer ring, `fR * 0.99`, tangential w/ bottom-half flip)
7. Draw hour numerals (CoreText, Jost-Regular, radius `fR * 0.65`)
8. Draw hour + minute hands (border layer → metallic gradient → highlight line)
9. Draw second hand if enabled (tail + counterweight circle + needle)
10. Draw center hub (5 concentric circles)

Scale factor: `scale = min(width, height) / clockSize` — all dimensions scale by this.

## What Still Needs Work

- **Typography**: Real v2 uses a custom in-house typeface. Jost is an approximation.
- **Hand geometry**: Bezier curves are close but real hands have more refined tapering (possibly concave edges).
- **Color accuracy**: All 16 palettes approximated from marketing thumbnails, not extracted from the binary.
- **Settings UI**: A native `configureSheet` (Options…) now exists; still plainer than the real app's polished sheet.
- **Languages**: Real app supports 10 languages (EN, FR, DE, ES, IT, JA, ZH, RU, TR, AR). Not implemented.
- **Code signing**: Needs Developer ID + notarization for distribution.

### Done (recent work)
- Outer minute-numeral ring with upright bottom half (was the "60 numeral" item).
- Day/Night auto-switching via `appearance: system`.
- `configureSheet` + cross-process live refresh (sandbox-correct settings on Sequoia).
- Hour-numeral / capsule geometry retuned so numerals no longer overlap the indices.
- Grain texture cached as a `CGImage` (perf).
